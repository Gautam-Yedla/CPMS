# CPMS Backend Documentation

## 1. Introduction
The Backend of the Car Parking Management System (CPMS) serves as the central orchestration layer. It manages the communication between the user-facing Frontend, the Supabase database, and the specialized Machine Learning (ML) services. It is responsible for business logic execution, data validation, and security enforcement across the system.

---

## 2. Architecture
The backend is built as a **Node.js** application using **Express.js v5** and **TypeScript**. 
- **API First**: The system follows a RESTful API architecture.
- **Middleware Pattern**: Utilizes Express middleware for a decoupled and extensible request pipeline (e.g., logging, security headers, authentication).
- **Service-Controller Pattern**: Controllers handle HTTP-level logic, while business logic is encapsulated within specific route modules and services.

---

## 3. Technology Stack

### Core Technologies
| Component | Technology |
| :--- | :--- |
| **Runtime** | Node.js (with Type: Module) |
| **Framework** | Express.js 5.x |
| **Language** | TypeScript for type safety |
| **Gateway** | Supabase JS (Database / Auth integration) |

### Functional Dependencies
- **Processing**: `fluent-ffmpeg` and `ffmpeg-static` for handling camera RTSP and HLS streams.
- **File Handling**: `multer` for managing binary data and media uploads from cameras.
- **Communication**: `axios` for internal and external service requests.
- **Security**: `helmet` (Security headers), `cors` (Cross-Origin Resource Sharing), and `morgan` (HTTP logging).

---

## 4. API Architecture & Strategy
The backend exposes a suite of modular API endpoints under the `/api` prefix:

| Prefix | Function |
| :--- | :--- |
| `/api/auth` | User login, registration, and session management. |
| `/api/user` | Profile management and user-specific data. |
| `/api/cameras` | Configuration and metadata management for hardware cameras. |
| `/api/ml` | Real-time monitoring of ML pipeline status and detection summaries. |
| `/api/stream` | Stream processing logic for live camera feeds. |
| `/api/permits` | Parking authorization and spot allocation logic. |
| `/api/violations` | Infraction reporting and management. |

---

## 5. Core Services & Logic

### Authentication & Authorization
The system uses a custom `authMiddleware` that integrates directly with Supabase's JWT system. It extracts the session token, validates the user identity, and attaches the user profile (including roles) to the request object.

### Machine Learning Integration
The backend serves as a bridge for the Python-based ML services:
- **Status Monitoring**: Reads the `status.json` generated by the ML pipeline to provide real-time health checks to the frontend.
- **Data Synchronization**: Synchronizes detection results with the Supabase database to ensure persistent storage of security events.

### Camera & Stream Management
Utilizing **FFmpeg**, the backend can process and reroute media streams. It manages camera metadata (location, connection status, RTSP URLs) and provides diagnostic tools for camera connectivity.

---

## 6. Database Integration
The backend communicates with the **PostgreSQL** database via the Supabase Client. 
- **Server-Side Privileges**: Certain administrative tasks (e.g., deleting cameras, updating violations) are gated by backend role checks (`admin`, `security`) before database interactions.
- **Triggers & Functions**: The backend works in tandem with database-level triggers (like `handle_new_user`) to ensure data consistency.

---

## 7. Security Implementation
- **Data Privacy**: Enforces Role-Based Access Control (RBAC) at the route level.
- **HTTP Security**: Uses `helmet` to set secure HTTP headers and protect against common web vulnerabilities.
- **CORS**: Strictly configured to allow requests only from authorized frontend domains.

---

## 8. Performance & Scalability
- **Asynchronous I/O**: Leveraging Node's non-blocking architecture for high-concurrency stream handling.
- **Optimized Data Retrieval**: Efficiently parses JSON-based ML outputs to minimize latency in dashboard updates.
- **TypeScript Optimization**: Compiles to optimized JavaScript for production deployment.

---

## 9. Conclusion
The CPMS Backend is the "brain" of the operation, ensuring that high-speed ML data and mission-critical parking information are handled securely and efficiently. Its robust TypeScript implementation and modular design make it a stable foundation for the overall parking management ecosystem.
